过山车

这是一道经典的二分匹配题，可用匈牙利算法求解。
那就用该题的样例讲讲匈牙利算法。

Sample Input
6 3 3
1 1
1 2
1 3
2 1
2 3
3 1
0

三男三女，左边为女，右边为男，有连线代表女愿意和男坐过山车，关系图如下所示：

先看1号女生，她愿意和1，2，3号坐过山车，那按照顺序，1号男生暂未匹配，就和1号男生匹配上了

再看2号女生，她愿意和1，3号坐过山车，那按照顺序，就试着和1号男生匹配，发现1号男生已经匹配好了
那就试着让和1号男生匹配好的1号女生换个人选，1号女生就去找2号男生，发现2号男生暂未匹配，1号女生就和2号男生匹配上了
那么这时1号男生就暂未匹配了，2号女生就和1号男生匹配了

最后看3号女生，她愿意和1号男生坐过山车，那按照顺序，就试着和1号男生匹配，发现1号男生已经匹配好了
那就试着让和1号男生匹配好的2号女生换个人选，2号女生就去找3号男生，发现3号男生暂未匹配，2号女生就和3号男生匹配上了
那么这时1号男生就暂未匹配了，3号女生就和1号男生匹配了

匈牙利算法注重的是一个让字。先选的会让着后选的，但也只有先选的能再选其他人，我才会让给你哦😯

用line[x][y]来表示连线，line[x][y]==1说明x与y可以匹配
用boy[x]来表示x号男生匹配到的女生是几号，初始化为0
用used[x]来表示当前这一轮x号男生匹配到的女生是几号，每轮都会初始化为0（先选的会让着后选的）
